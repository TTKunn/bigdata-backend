## 项目概述

### 项目背景

本项目是一个基于大数据技术的分布式商城后端系统，旨在通过Hadoop生态系统、HBase和Redis等大数据技术，实现高并发、高可用、海量数据处理的电商平台后端服务。该系统将充分发挥大数据技术在数据存储、实时计算、缓存优化等方面的优势，解决传统关系型数据库在高并发场景下的性能瓶颈问题。

### 项目目标

1. **技术验证目标**：验证大数据技术在电商场景下的应用可行性
2. **性能目标**：支持高并发访问，实现实时数据处理
3. **数据目标**：处理海量商品、订单、用户数据
4. **业务目标**：提供完整的商城后端服务功能

### 项目范围

#### 包含内容
- 商品管理模块（基于HBase存储）
- 订单处理模块（基于HBase+Redis）
- 用户管理模块（基于HBase+Redis）
- 销售分析模块（基于HBase+Redis）
- 购物车管理（基于Redis）
- 库存管理（基于Redis+HBase）

#### 不包含内容
- 用户注册/登录功能（默认单用户）
- 商品CRUD管理（除创建接口外的完整管理功能，仅开发环境提供创建接口用于数据准备）
- 前端界面开发
- 支付网关集成
- 物流系统集成

## 功能需求分析

### 核心业务流程

#### 1. 商品浏览流程
```
用户访问商城 → 商品列表页 → 商品详情页 → 加入购物车 → 购物车结算
```

#### 2. 订单处理流程
```
购物车结算 → 创建订单 → 库存扣减 → 订单确认 → 支付处理 → 订单完成
```

#### 3. 销售统计流程
```
订单完成 → 实时统计更新 → 离线数据分析 → 统计面板展示
```

### 功能模块详细需求

#### 1. 商品管理模块

**简化说明**：本系统侧重大数据技术的学习和应用，不实现完整的商品管理功能，仅提供数据存储和API查询能力。商品信息主要通过批量导入方式预存到HBase数据库中，前端可通过API查询商品信息。为方便开发和测试，提供商品创建接口（仅开发环境使用，不在正式环境暴露）。

**1.1 商品数据存储**
- **功能描述**：商品信息预存到HBase，提供基础查询和开发用创建API
- **数据存储**：HBase (product_info表)
- **功能点**：
  - 商品信息批量预导入（初始化数据）
  - 商品信息查询API（按ID、分类、品牌等）
  - 商品状态查询（上架/下架状态）
  - 商品创建API（仅开发环境，方便测试和数据准备）

**1.1.1 商品图片存储方案**
- **存储架构**：HDFS分布式文件存储 + HBase元数据管理
- **HDFS存储路径**：`/product_images/{category_id}/{product_id}/{timestamp}_{filename}`
- **HBase元数据存储**：在product_info表的cf_detail列族中存储单张图片信息JSON对象
- **图片限制**：每个商品仅支持上传单张主图，最大不超过10MB
- **图片信息结构**：
  ```json
  {
    "id": "hdfs://bigdata01:9000/product_images/0001/00000001/1640995200000_main.jpg",
    "type": "main",
    "size": 245760,
    "upload_time": "2024-01-01 12:00:00"
  }
  ```
- **技术优势**：
  - HDFS提供高可用、高容错的分布式存储
  - 支持PB级图片数据存储和自动备份
  - HBase提供快速的元数据查询
  - 支持JPEG/PNG等多种格式，单图片最大10MB
- **访问方式**：
  - 通过REST API获取图片元数据
  - 通过HDFS HTTP接口或自定义服务访问图片文件
  - 支持CDN加速和缓存优化

**1.2 商品库存管理**
- **功能描述**：实时管理商品库存，支持高并发扣减
- **技术实现**：Redis缓存 + HBase持久化
- **功能点**：
  - 库存实时查询
  - 库存原子扣减
  - 库存同步机制
  - 库存预警机制

#### 2. 订单处理模块

**2.1 订单创建**
- **功能描述**：基于购物车创建订单
- **数据存储**：HBase (order_history表)
- **功能点**：
  - 订单号生成（分布式唯一）
  - 订单信息校验
  - 库存预扣检查

**2.2 订单状态管理**
- **功能描述**：管理订单生命周期
- **技术实现**：Redis缓存状态 + HBase持久化
- **状态流转**：
  - 待付款 → 已付款 → 待发货 → 已发货 → 已完成
  - 任意状态 → 已取消

**2.3 订单查询**
- **功能描述**：支持多维度订单查询
- **查询维度**：
  - 按用户ID查询
  - 按订单状态查询
  - 按时间范围查询
  - 按订单金额查询

#### 3. 用户管理模块

**简化说明**：本系统侧重大数据技术的学习和应用，为简化系统复杂度，仅使用一个默认用户。该用户既是前台普通用户，也是后台管理用户，无需进行权限校验、用户注册、登录认证、会话管理等功能。系统启动时自动创建默认用户，所有操作都以该用户身份执行。

**3.1 默认用户管理**
- **功能描述**：系统使用单一默认用户，无需用户管理功能
- **数据存储**：HBase (user_profile表)
- **功能点**：
  - 系统启动时自动创建默认用户
  - 默认用户信息查询API
  - 用户统计信息自动维护（订单数、消费金额等）

**3.2 权限与会话简化**
- **说明**：系统无需权限校验和会话管理，所有API调用都使用默认用户身份
- **设计考虑**：
  - 无需登录认证机制
  - 无需权限控制逻辑
  - 无需会话状态管理
  - 所有业务操作直接使用默认用户ID

#### 4. 购物车管理模块

**4.1 购物车操作**
- **功能描述**：管理用户的购物车信息
- **数据存储**：Redis Hash结构
- **功能点**：
  - 添加商品到购物车
  - 修改商品数量
  - 删除购物车商品
  - 清空购物车
  - 购物车商品查询

**4.2 购物车数据结构**
```
Key: cart:{userId}
Field: product_{productId}
Value: {"quantity": 数量, "add_time": 添加时间, "selected": 是否选中}
```

#### 5. 销售分析模块

**5.1 实时销售看板**
- **功能描述**：实时展示销售数据统计
- **数据来源**：Redis缓存
- **统计指标**：
  - 今日销售额
  - 今日订单数
  - 今日访客数
  - 平均客单价
  - 热销商品排行

**5.2 历史销售分析**
- **功能描述**：基于历史数据的销售分析
- **数据来源**：HBase存储的历史数据
- **分析维度**：
  - 按日/周/月统计
  - 按商品分类统计
  - 按地区统计
  - 销售趋势分析

**5.3 排行榜功能**
- **功能描述**：实时更新各类排行榜
- **技术实现**：Redis Sorted Set
- **排行榜类型**：
  - 销量排行榜（日/周/月）
  - 销售额排行榜
  - 新品排行榜
  - 收藏排行榜

### 数据同步策略

#### 1. Redis到HBase同步
- **同步时机**：定时同步 + 触发同步
- **同步内容**：
  - 库存数据同步
  - 订单状态同步
  - 销售统计同步
- **同步机制**：
  - 消息队列异步处理
  - 批量同步优化
  - 失败重试机制

#### 2. 缓存更新策略
- **Cache-Aside模式**：先更新数据库，再删除缓存
- **Write-Through模式**：同时更新缓存和数据库
- **缓存过期策略**：基于时间 + 基于访问

#### 3. 数据一致性保障
- **最终一致性**：通过定时任务修复不一致
- **强一致性**：使用分布式锁保证重要操作
- **监控告警**：数据一致性检查和告警

## 技术需求分析

### 性能需求

#### 1. 并发处理能力
- **目标并发量**：1000+ TPS（每秒事务数）
- **库存扣减并发**：支持100+并发扣减同一商品
- **数据查询响应**：平均响应时间 < 100ms
- **订单创建响应**：平均响应时间 < 200ms

#### 2. 数据处理能力
- **商品数据量**：10万+ 商品信息
- **订单数据量**：100万+ 订单记录（模拟数据）
- **用户数据量**：10万+ 用户信息
- **商品图片存储**：HDFS分布式存储，每个商品支持单张主图
- **图片处理能力**：支持JPEG/PNG等多种格式，单图片最大10MB
- **实时统计**：支持实时更新和查询

### 可靠性需求

#### 1. 数据持久化
- **Redis数据持久化**：RDB + AOF双重保障
- **HBase数据持久化**：HDFS三副本存储
- **数据备份策略**：定期全量备份 + 增量备份

#### 2. 高可用性
- **集群部署**：一主两从架构
- **故障转移**：自动故障检测和恢复
- **服务降级**：核心功能优先保障

### 安全性需求

#### 1. 数据安全
- **数据加密**：敏感数据加密存储
- **访问控制**：基于角色的访问控制
- **SQL注入防护**：参数化查询防护

#### 2. 系统安全
- **网络安全**：防火墙配置
- **日志审计**：完整的操作日志
- **异常监控**：实时异常检测

- category_id：分类ID（4位）

- timestamp：时间戳（13位）

**列族结构**：
```
product_info

│   ├── description: 描述

│   ├── total_stock: 总库存
```

### Redis数据结构设计

#### 1. 商品库存缓存
```redis
# 商品库存
SET stock:{product_id} {quantity} EX 3600

# 秒杀库存（可选）
SET seckill_stock:{activity_id}:{product_id} {quantity}
```

#### 2. 用户购物车
